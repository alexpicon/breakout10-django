This is a small Django application built to demonstrate the Model–View–Template (MVT) pattern, Django's take on classic MVC.

How the layers map

  • Model — main/models.py defines a Message with a single text field, persisted to SQLite.
  • View — main/views.py contains the request-handling functions: home lists every message, about renders this page.
  • Template — main/templates/main/about.html is this HTML file, rendered by the about view.
  • Controller / URL routing — mysite/urls.py wires URL paths to views.
